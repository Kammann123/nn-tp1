1. Descripción del problema¶

Se desea armar un clasificador de correos electrnicos o noticias en funcin del tpico sobre el cual trata dicho correo electrnico o noticia, para ello, se modela el problema considerando una variable aleatoria discreta $X$ con distribucin categrica de $k_x = 20$ categoras, siendo estas los tpicos posibles que pueden tratar los documentos clasificados.

$$X \sim Categorica(p_1, p_2, ..., p_{k_x})$$

Las caractersticas a analizar de los documentos para la clasificacin, son las palabras. Se asume que el orden de las palabras no importa, que son independientes entre s y que se encuentran identicamente distribudas. De esta forma, se define una segunda variable aleatoria discreta $Y$ con distribucin categrica de $k_y$ categoras, siendo estas las posibles palabras de nuestro vocabulario.

$$Y \sim Categorica(p_1, p_2, ..., p_{k_y})$$

2. Esquema de clasificación¶

El modelo de clasificacin es probabilstico, utilizando el criterio de mximo a posteriori, es decir se busca encontrar cul es el tpico para el cual la probabilidad de que sea ese tpico sea mxima, dado la informacin que tenemos del mail o noticia.

$$P(y|x_1,...,x_n)$$

Aplicando la regla de Bayes,

$$P(y|x_1,...,x_n) = \frac{P(x_1,...,x_n|y) \cdot P(y)}{P(x_1,...,x_n)}$$

Luego, computacionalmente puede resultar demasiado compejo y costoso obtener las funciones de masa de probabilidad conjunta para las $n$ palabras dentro de un texto, dado cada posible tpico. Obtener tales funciones es un proceso de estimacin, por lo tanto incluso si computacionalmente se pudiera estimar esa probabilidad conjunta, la cantidad de datos necesarios para poder estimarla con poco error resulta muy grande dada la cantidad de variables includas. Con lo cual, con no muchos datos tambin habra un error mayor en la prediccin.

Para esto, se emplea un clasificador naive bayes, donde la condicin naive o ingenuo impone independencia de las variables. De esta forma, la verosimilitud se aproxima y la expresin resulta como se muestra a continuacin,

$$P(y|x_1,...,x_n) \approx \frac{P(x_1|y) \cdot ... \cdot P(x_n|y) \cdot P(y)}{P(x_1,...,x_n)}$$

El denominador de esta expresin es invariante a las categoras de noticias o tpicos, es decir, es un nmero que se emplea por igual para cada probabilidad a posteriori. Entonces, como el proceso de clasificacin slo busca ver cul de estas expresiones resulta mayor, si todas estn afectadas por un factor idntico, removerlo no afecta,

$$P(y|x_1,...,x_n) \approx \left[ \prod_{i=1}^{n} P(x_i|y) \right] \cdot P(y)$$

En conclusin,

$$\hat{y} = argmax_y \left[ \left[ \prod_{i=1}^{n} P(x_i|y) \right] \cdot P(y) \right]$$

3. Parámetros del modelo¶

A modo de conclusin del anlisis previo que describe el esquema de clasificacin a emplear, se pueden determinar cuales son los parmetros del modelo que son necesarios estimar mediante el conjunto de datos de entrenamiento para tener listo el clasificador para predecir resultados.

  2. $P(x_i|y)$, es la distribucin de la variable categrica palabra, condicionada al tpico.

8. Estimación de distribución de probabilidad condicional¶

Para la estimacin, se cuentan las apariciones de cada palabra dado el tpico de las noticias y luego se normaliza esa frecuencia absoluta para obtener la frecuencia relativa.

8.1. Overfitting¶

El overfitting es un problema que ocurre cuando el modelo utilizado no es capaz de predecir correctamente frente a datos nuevos que no se encontraban en el conjunto de entrenamiento, lo cual le quita capacidad de generalizar, un aspecto fundamental en la resolucin de problemas de inteligencia artificial. En este caso, para resolver el problema descripto se propone emplear un clasificador naive bayes multinomial, que puede traer apareado un problema de overfitting que se ilustra a continuacin.

Sea un texto compuesto por $N$ palabras, para clasificarlo se busca la mxima probabilidad a posteriori, es decir aquella categora $y$ para la cual la probabilidad de que ese texto pertenezca sea mxima a comparacin de otras categoras,

$$P(y|x_1,...,x_N)=\frac{P(x_1,...,x_N|y) \cdot P(y)}{P(x_1,...,x_N)}$$

Aplicando naive bayes, se asumen que las palabras son independientes y luego se aade que sean idnticamente distruidas segn el tipo de problema que nos proponemos resolver, entonces

$$P(y|x_1,...,x_N)=\frac{P(x_1|y) \cdot ... \cdot P(x_N|y) \cdot P(y)}{P(x_1,...,x_N)}$$

En funcin de cmo se estima la distribucin de probabilidades, una palabra $x_1$ que no apareci en una categora durante el entrenamiento fue ponderada con una probabilidad nula, lo cual hace que todo el producto se anule, y si la probabilidad fuera muy grande, por culpa de esta nica palabra toda la probabilidad a posteriori se anula llevando a una prediccin errnea. Para solucionar esto, se propone emplear el mtodo de Laplacian Smoothing.

8.2. Laplacian Smoothing¶

El mtodo de Laplacian Smoothing consiste en agregar un cierto conocimiento a priori dentro de la distribucin estimada, de forma tal que cuando no se posee demasiada evidencia durante el entrenamiento, la informacin aportada por tal conocimiento a priori nos permite evitar caer en el escenario de las probabilidades nulas.

El procedimiento consiste en fijar el valor de un hiperparmetro $\alpha$, que se suma como cantidad de ocurrencias a cada palabra durante el anlisis de frecuencias absolutas en la etapa de estimacin. As, se introduce un conocimiento a priori de cantidad de ocurrencias igual para todas las palabras, es decir, estamos diciendo que sin informacin supondramos que la distribucin es uniforme. Por lo general, la magnitud de este parmetro define qu tanta informacin de entrenamiento se necesita para que deje de valer dicho conocimiento a priori.

11. Clasificación por máximo a posteriori¶

Como ya se explic anteriormente, el esquema de clasificacin consiste en buscar el mximo a posteriori. Para ello, se busca calcular

$$P(y|x_1,...,x_n) \approx \left[ \prod_{i=1}^{n} P(x_i|y) \right] \cdot P(y)$$

En este caso, por cada palabra que aparezca, multiplicamos por su probabilidad condicionada tantas veces como aparezca durante la productoria. Esto implica, que podramos modificar el orden de la operacin iterando sobre las palabras del vocabulario,

$$P(y|x_1,...,x_n) \approx \left[ \prod_{i=1}^{m} P(v_i|y)^{\# v_i} \right] \cdot P(y)$$

Es decir, se hace la productoria sobre las palabras del vocabulario $v_i$, y cada una de ellas se encuentra elevada a la cantidad de veces que aparece esa palabra dentro del texto, es decir, $\# v_i$.

Entonces, en la prctica se tiene,

  • Una matriz que contiene para cada tpico (filas) las probabilidades de cada palabra (columnas), es decir con una dimensin de k x m.
  • Un vector con las probabilidades a priori, es decir de dimensin k x 1.
  • Un vector con la cantidad de ocurrencias de cada palabra de un documento, es decir, de dimensin k x 1.

As, se eleve la matriz al vector de ocurrencias y luego se ejecuta el producto punto con el vector de probabilidades a priori. Finalmente, se obtiene con esto un vector con el valor numrico para comparar cada categora.

11.1 Problema numérico¶

Si el preprocesamiento no se vuelve lo suficientemente estricto y no limita el vocabulario, la cantidad de palabras provoca que los valores numricos obtenidos en la verosimilitud sean demasiado chicos y empiezan a darse errores numricos por la precisin del punto flotante empleado por NumPy.

Este problema se encontr comparando este modelo con el de log probabilidades, y se pudo observar que si se escala la distribucin (no se la normaliza), entonces el problema numrico desaparece.

Clasificación utilizando log probabilidades¶

Si consideramos que el denominador de la expresin del mximo a posteriori es un trmino invariante a las categoras, por lo cual lo removemos, y luego aplicamos el logaritmo natural, obtenemos una expresin de la siguiente forma,

$$ln[P(y|x_1,...,x_N)] \propto ln[P(x_1|y)] + ... + ln[P(x_n|y)] + ln[P(y)]$$

Luego, si aplicamos esto mismo sobre la otra forma de ejecutar la probabilidad a posteriori, donde en vez de iterar sobre las palabras del texto se itera sobre las palabras del vocabulario, entonces esta misma operacin queda,

$$ln[P(y|x_1,...,x_N)] \propto \# v_1 \cdot ln[P(v_1|y)] + ... + \# v_n \cdot ln[P(v_n|y)] + ln[P(y)]$$

Entonces, matemticamente se puede escribir como el producto punto entre un vector con las ocurrencias de las palabras en el texto y un vector con las log probabilidades de cada palabra, y finalmente sumarle el vector con las log probabilidades a priori.

Exactitud¶

La exactitud se define como la relacin entre el nmero de predicciones correctas y el nmero total de predicciones. Es decir, es la proporcin de predicciones realizadas que fueron acertadas.

$$Accuracy = \frac{TP+TN}{TP+FP+TN+FN}$$

En esta expresin, se usan abreviaciones para los siguientes trminos,

  • TP, True Positive, aquellas predicciones positivas que verdaderamente fueron positivas
  • TN, True Negative, aquellas predicciones negativas que verdaderamente fueron negativas
  • FP, False Positive, aquellas predicciones positivas que en realidad eran negativas
  • FN, False Negative, aquellas predicciones negativas que en realidad eran positivasfrom sklearn import metrics

Esta mtrica no es apropiada si las clases estn desbalanceadas. Se debe comparar siempre con la solucin trivial, que es asumir que el resultado de todas las predicciones es la clase mayoritaria. Por otro lado, tampoco es correcto usar la exactitud cuando por ejemplo se busca minimizar el nmero de falsos negativos (es el caso del diagnstico de enfermedades).

Precisión¶

La precisin se define como la relacin entre positivos verdaderos y la suma de todos los clasificados como positivo (es decir, se suman los verdaderos positivos con los falsos positivos). En otras palabras, esta mtrica informa de cierto modo la calidad de una deteccin del modelo, ya que indica la proporcin de positivos detectados que efectivamente fueron positivos.

$$Precision = \frac{TP}{TP+FP}$$

Sensibilidad¶

F1 Score¶

El F1 score es una mtrica que permite vincular la precisin y la sensibilidad, de la siguiente forma.

$$F1 = 2 \cdot \frac{Precision \cdot Recall}{Precision + Recall}$$

El valor F1 asigna la misma importancia a la precisin y la exhaustividad.
